# Tomato Clock 番茄钟网页应用PRD
## 一、功能需求
- 倒计时功能：支持开始、暂停、重置操作，默认工作25分钟，休息5分钟。
- 自定义时长：用户可以自定义工作和休息的时长，范围为1-60分钟。
- 通知提醒：时间到了要发出通知提醒，用户可以选择是否播放提示音。
- 统计功能：记录完成的番茄钟数量，智能生成统计报告，帮助用户了解自己的工作效率。

## 二、核心流程
1. 经典番茄钟流程
- 工作时段：25 分钟（默认，应支持自定义）
- 短休息：5 分钟
- 长休息：15-30 分钟（每完成 4 个番茄后）
- 循环计数：自动记录已完成番茄数，达到 4 个后提示长休息

2. 计时器基本操作
- 开始/暂停：点击大按钮切换状态
- 重置：将当前阶段时间重置为初始值
- 跳过当前阶段：允许用户手动进入下一阶段（例如想提前结束休息）

3. 状态切换与提示
- 自动切换：计时结束时自动进入下一阶段（工作→短休息→工作…）
- 视觉区分：工作时段用暖色（如红色/橙色），休息时段用冷色（如绿色/蓝色）
- 通知提醒：使用 Notification API 在后台发出系统通知（结合声音提示）

4. 任务管理
- 待办列表：添加任务，点击任务旁的“番茄”图标开始专注该任务
- 今日完成数：每个任务旁显示已完成的番茄数

## 三、界面设计
1. 主界面布局（简洁至上）
中央大计时器：数字巨大，字体清晰（建议使用等宽字体）
环形进度条：直观展示当前阶段剩余比例，颜色与当前阶段一致（如橙红/蓝绿），环形进度条在外，环内是当前剩余时间
阶段名称：当前是“专注”还是“休息”
已完成番茄数：小图标显示今日已完成个数（如 🍅 × 3）
底部按钮区：开始/暂停、重置、跳过（尽量用图标+文字）
每个阶段倒计时结束时有提示音，同时有系统通知提醒用户（会事先向用户请求权限）。

2. 色彩与视觉反馈
工作模式：主色为橙红，营造紧迫感
休息模式：主色为蓝绿，放松感
按钮状态：悬停/点击有微动效，禁用状态半透明
暗色模式：提供开关，保护夜间使用眼睛

3. 响应式适配
移动端：按钮足够大（至少 44×44 触控区域），计时器字号响应式
桌面端：可适当增加侧边栏（如任务列表、统计）

4. 动效与细节
计时结束时有微震动（如果支持 navigator.vibrate）
切换阶段时平滑过渡（如背景色渐变）
按钮按下效果（缩小再恢复）

## 四、其他考虑
清晰的状态指示：一眼就能看出现在是工作还是休息，还剩多久。
避免误操作：重置和跳过应有点击确认或二次按钮（如长按重置）。
温柔的引导：首次使用时，可显示简短的引导浮层，解释番茄钟用法。
尊重用户：不要强制用户必须完成番茄，允许随时暂停或放弃。
